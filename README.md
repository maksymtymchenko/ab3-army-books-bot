# Army Books Telegram Bot (Node.js + TypeScript)

Telegram бот для керування бібліотекою та бронюванням книжок на основі існуючого бекенду (`/api/books`, `/api/reservations`).

## 1. Підготовка оточення

1. Встанови залежності:

```bash
npm install
```

2. Створи файл `.env` на основі прикладу:

```bash
cp .env.example .env
```

3. Заповни змінні в `.env`:

- `BOT_TOKEN` – токен твого Telegram-бота (від `@BotFather`).
- `BASE_API_URL` – базовий URL до бекенду, наприклад: `http://localhost:3000`.

## 2. Запуск бота

- Режим розробки (TypeScript напряму):

```bash
npm run dev
```

- Продакшн (спочатку зібрати, потім запустити зібраний JS):

```bash
npm run build
npm start
```

## 3. Основні можливості бота

- **Додавання книги**
  - Команда: `/addbook` або кнопка **«➕ Додати книгу»**.
  - Бот поетапно запитає:
    - назву книги;
    - автора;
    - `coverUrl` (URL обкладинки);
    - короткий опис (можна пропустити, надіславши `-`).
  - Використовує ендпоінт `POST /api/books`.

- **Видалення книги**
  - Команда: `/deletebook` або кнопка **«🗑 Видалити книгу»**.
  - Бот попросить ввести `ID` книги.
  - Використовує ендпоінт `DELETE /api/books/{id}`.

- **Робота з замовленнями (бронюваннями)**
  - Команда: `/orders` або кнопка **«📚 Замовлення»**.
  - Бот підтягує список замовлень зі статусом `pending` через `GET /api/reservations`.
  - Для кожного замовлення показує:
    - `id` замовлення;
    - `bookId`;
    - статус;
    - дату створення.
  - Під кожним замовленням додаються інлайн-кнопки:
    - **«✅ Прийняти»** – оновлює статус на `confirmed` (`PATCH /api/reservations/{id}/status`);
    - **«❌ Відхилити»** – оновлює статус на `rejected` (`PATCH /api/reservations/{id}/status`).

## 4. Файли, що відповідають за логіку

- `src/api.ts` – Axios-клієнт для звернень до бекенду:
  - `createBook`, `deleteBook`, `listReservations`, `updateReservationStatus`.
- `src/bot.ts` – основна логіка Telegram-бота:
  - команди `/start`, `/addbook`, `/deletebook`, `/orders`;
  - Wizard-сцени для додавання та видалення книг;
  - інлайн-кнопки для підтвердження / відхилення замовлень.
